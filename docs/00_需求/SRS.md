# SRS - 研发管理平台（方案B）

## 1. 范围
- 覆盖需求管理、研发管理、CI 管理、自动化测试、Bug 管理、版本发布与审计。

## 2. 功能需求（摘要）
- 需求管理：PRD/用户故事/验收标准/状态流转。
- 研发管理：任务拆解、分支策略、MR/PR、代码评审、研发统计。
- CI 管理：流水线状态、失败定位、构建产物管理。
- 自动化测试：用例库、测试计划、测试报告、覆盖率/趋势。
- Bug 管理：提交/分派/复现/修复/验证/回归。
- 版本发布：版本号、变更清单、灰度/回滚、发布审批、发布记录。

## 3. 数据与对象
- 需求：PRD、用户故事、验收标准、状态。
- 研发：任务、分支、MR/PR、评审记录。
- CI：流水线、步骤、日志摘要、构建产物。
- 测试：用例、计划、报告、覆盖率与趋势。
- Bug：缺陷、严重级别、状态流转、回归记录。
- 发布：版本号、变更清单、审批记录、发布记录、回滚记录。

## 4. 接口与集成
- CI 数据源：Jenkins/GitHub Actions（初期限定，后续扩展）。
- 产物存储：统一产物索引与下载入口。

## 5. 权限与审计
- 角色：Boss/PM/RD Lead/Engineer/QA/Viewer。
- 权限控制：模块级与操作级权限。
- 审计日志：关键变更与审批记录需可追踪。

## 6. 约束与质量
- 性能：分页查询 < 2s。
- 稳定性：核心流程可用性 99.9%。
- 可测试性：状态机与流程可通过用例验证。

## 7. 验收与可追溯性
- 需求-任务-测试-发布全链路可追踪。
- 每个关键功能对应验收标准与用例编号。
